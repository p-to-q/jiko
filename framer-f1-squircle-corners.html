Squircle Corner Study

Framer F1 keyboard corner analysis & implementation

The problem

CSS border-radius creates a circular arc that meets the straight edge at a tangent point. At that junction, curvature jumps discontinuously from 0 (flat) to 1/r (arc) — a G1 join. Your eye detects this as a subtle "pinch" where the curve begins. A squircle (superellipse n=4) uses one continuous equation: curvature flows smoothly from 0 at the midpoint of each side to maximum at the corner. This G2 continuity is what makes Apple's icons, Figma's corner smoothing, and the Framer F1 feel "right."

Implementation code

Method 1: CSS corner-shape CSS

Chrome 139+. Progressive enhancement — falls back to border-radius in unsupported browsers.

/* Native CSS squircle — Chrome 139+ */ .squircle { border-radius: 24px; corner-shape: squircle; } /* Custom exponent */ .superellipse { border-radius: 24px; corner-shape: superellipse(3); }

Method 2: SVG clip-path SVG

Cross-browser. Pre-compute the path and clip any element.

/* Use the smoothRectPath() function */ const path = smoothRectPath( 0, 0, // x, y 200, 80, // width, height 12, // corner radius 0.6 // smoothing (0–1) ); el.style.clipPath = `path('${path}')`;

Method 3: figma-squircle JS

NPM package. Same algorithm Figma uses internally.

import { getSvgPath } from 'figma-squircle'; const path = getSvgPath({ width: 200, height: 80, cornerRadius: 12, cornerSmoothing: 0.6 }); el.style.clipPath = `path('${path}')`;

Method 4: Superellipse formula SVG

Pure math. |x/a|^n + |y/b|^n = 1. Full control over exponent.

// Parametric superellipse function superellipse(t, a, b, n) { const c = Math.cos(t); const s = Math.sin(t); return [ a * Math.sign(c) * Math.pow(Math.abs(c), 2/n), b * Math.sign(s) * Math.pow(Math.abs(s), 2/n) ]; }
