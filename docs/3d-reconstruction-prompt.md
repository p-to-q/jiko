## 3. 核心尺寸规范

所有尺寸从 USB-C 母座宽度（8.34 mm）锚定推出。

```
scale = 8.34 mm ÷ 0.19 (model_usbW) ≈ 43.9 → 取整 44 mm/unit
```

**你拿到代码后第一条任务**：在 `ShowcaseStage.tsx` 里找到 `buildHardware()` 函数，验证 bodyW=1.82、bodyH=bodyW/(2/3)、bodyDepth=0.15，然后拿 calipers（或对比已知物体）在脑子里建立比例感。

代码中的精确尺寸（精确视觉版）：

| 参数 | 模型值 | 物理尺寸 |
|---|---|---|
| 单槽宽度 (slotW) | 0.010 | **0.44 mm** |
| 单槽高度 (slotH) | 0.200 | **8.8 mm** |
| 槽间距 (gap) | 0.022 | **0.97 mm** |
| 底部方孔 (sqSize) | 0.035 | **1.54 mm** |

布局从上到下：
1. 长槽 1 — 0.44 × 8.8 mm
2. 长槽 2 — 0.44 × 8.8 mm
3. 长槽 3 — 0.44 × 8.8 mm
4. 长槽 4 — 0.44 × 8.8 mm
5. 底部复合 — 短槽（0.44 × 8.8 mm）+ squircle 方孔（~1.54 mm）

阵列高度约占总左侧面高度的 40%。

位置：左侧平面 `x = -(bodyW × 0.5) - 0.003`，z 偏移 `bodyDepth × 0.1`
填孔材质：深色（`0x010101`）+ 边缘淡蓝灰高光（`0x9fb0c4`，AdditiveBlending 透明度 0.28）

**3D 打印版**：因为 0.44 mm 对加工/可视性过窄（文档也提示了），**打印版的槽宽强制改为 0.8–0.9 mm**。在 `print-notes.md` 中注明此改动。

## 4. 必须交付 4 个版本（每个版本独立输出）

### 版本 A — 渲染版（Render / Showcase）

**用途**：本地打开观看，视觉效果炸裂，能看见内部细节。**这个版本可以出多个子版本。**

| 子版本 | 风格要求 |
|---|---|
| **A1 — 暗夜金属** | 全黑机身 + 高光倒角 + 锐利反射，在暗背景下凸显轮廓。渲染时打 2–3 盏 HDRI 环境光 |
| **A2 — 半透视拆解** | 外壳半透明（glass/or 40% opacity），内部可见：屏幕背光层、USB-C 舌片、螺丝柱位、麦克风导音管示意。给用户看到"里面长什么样" |
| **A3 — 木纹实验** | 机身换成胡桃木纹理（Pilot S20 风格），金属部件保留。和 A1 形成材质对比 |
| **A4 — 白金极简** | 通体铂金/钛金属质感 + 拉丝表面，像一支高级自动铅笔。材质参数你自己调 |
| **A5 — 任意自由发挥** | 你觉得什么风格酷就做什么，材质、颜色、光效、氛围随便改 |

**输出格式**：
- 4K 渲染图（PNG/EXR，多角度：正面 30°、左侧 45°、右侧 45°、底部仰视、顶部俯视）
- 可选关键帧动画 360° 旋转 GIF
- 源文件（blend / fbx / 项目文件均可）

**技术要求**：
- 屏幕像素内容必须从机身表面**物理凸起**（0.1–0.3 mm 浮雕）
- 材质必须 PBR：metalness + roughness + clearcoat + env map
- Macro 特写镜头能看清麦克风孔和 USB-C 舌头

---

### 版本 B — HTML 版（Self-Contained Web Viewer）

**用途**：双击就能在浏览器打开，不需要任何工具或服务器。

**要求**：
- 单个 `.html` 文件，**可离线打开**
- 内嵌 Three.js（从 CDN 加载或 base64 内联），用 `<script type="importmap">` 或 `<script src="...">`
- 页面加载后自动展示 3D 模型，支持鼠标拖拽旋转/缩放
- 左上角或底部加一个简单的 UI 切换：在不同版本（A1–A5）之间切换材质/风格
- 被包含的 3D 模型用 **GLB 格式**，可以 base64 嵌入 HTML 或单独放入同目录
- 如果 GLB 太大无法内嵌，至少提供 `index.html + model.glb` 文件夹形式

**交互功能**：
- OrbitControls（旋转/平移/缩放）
- 版本切换下拉菜单或按钮组
- 点击部件可弹出标注（tooltip 显示部件名称）
- 暗色背景，干净 UI

**输出格式**：
- `jiko-one-viewer.html`（单文件，推荐）
- 或 `jiko-one-viewer/`（文件夹，含 index.html + model.glb）

---

### 版本 C — 3D 打印版（Physical Print）

**用途**：直接丢进 Bambu Studio / PrusaSlicer / Cura 就能打。

**要求**：
- **水密（watertight）**，STL 无破面、无反转法线
- **实体不厚度小于 0.8 mm**（FDM 打印下限），太薄的部件加厚
- **装配间隙 0.2 mm**：按钮和机身之间、螺丝和螺孔之间
- **不需要支撑就能打印**的朝向设计（或明确标注推荐打印方向）
- 分件输出：`body.stl`、`button.stl`、`screws.stl`、`backplate.stl`
- 也可以出一版合并的 `all-in-one.stl`
- 加一个 `print-notes.md`，写明：推荐打印参数（层高 0.12/0.2、填充 15%/100%、材料 PLA/PETG/树脂）、是否需要支撑、缩放比例建议

**可选**：
- 把"热"字散热标记做成真正的**镂空通孔**（散热功能）
- 麦克风和 USB-C 孔**真的打通**

**输出格式**：
- `*.stl`（首选）
- `*.3mf`（可选，Bambu/Prusa 项目文件更好）

---

### 版本 D — 最还原版（Design Reference / Industrial Design）

**用途**：给设计师和工业设计师看的黄金参考。这个版本要不计成本和面数，追求**与原始 Three.js 实现完全一致**。

**要求**：
- **每一个尺寸必须和 `ShowcaseStage.tsx` 代码完全一致**（包括那些 `0.002` 级别的微调偏移）
- 所有部件独立成体，有正确的装配层次结构
- 保留全部倒角（bevel）和微倒角
- 输出 **CAD 可编辑格式**（.step/.iges），让工业设计师能直接导入 Fusion 360 / SolidWorks / Rhino
- 面部导出：标注每个面的目标材质/颜色/表面处理（喷砂、拉丝、光面）
- 输出一份 **尺寸校验报告**：逐项核对代码值 vs 模型值的偏差，偏差超过 0.05 mm 的必须解释原因
- 导出动画/爆炸视图（exploded view），展示装配顺序

**可选项**：
- 内部堆叠示意：电池、主板、屏幕模组的简化体块（用不同颜色半透明表示）
- 螺丝规格标注（M1.2、M1.6等）

**输出格式**：
- `*.step` / `*.iges`
- 爆炸视图 PNG/PDF
- 尺寸校验 PDF/表格

## 5. 建模硬性要求（所有版本通用）

### 必须实打实建模

| 部件 | 最低要求 |
|---|---|
| **机身** | Squircle 形状，带倒角 bevel。主体材料黑色金属质感 |
| **屏幕** | 屏幕内容有**物理凸起**（0.1–0.3 mm 微量浮雕），渲染时肉眼可见 |
| **麦克风** | **真正挖通一个洞**。从上往下：边缘亮环（chamfer）→ 沉孔锥面（countersink）→ 内孔（bore），三层结构 |
| **USB-C** | **真正挖出接口**。外凹坑 → USB-C 口 → 内部舌头（tongue，偏下位于口内）→ 正面 rim → 正面 mouth。不能是平面贴图 |
| **螺丝** | 背面 2 颗真实螺丝（对角布置），圆柱+沉头+十字槽 |
| **侧边按钮** | 按钮实体 + 机身侧的导轨凹槽，两者分离 |
| **散热孔** | 左侧面竖槽阵列：4 条长竖槽 + 底部复合（短槽 + squircle 方孔）。打印版槽宽加宽到 0.8–0.9 mm |
| **背板** | 背面 Squircle 面板，与主体分离 |

## 8. 冲突处理与决策原则

### 8.1 优先级

1. **视觉忠实度 > 制造可行性**（目前是工业设计展示阶段）
2. **与 ShowcaseStage.tsx 一致 > 重新发明**（代码是第一参考）
3. **先做对再做快 > 一次性完美**
4. **多版本输出 > 单版本死磕**

### 8.2 冲突处理表

| 场景 | 处理方式 |
|---|---|
| 文档 vs 代码 | **以代码为准** |
| 尺寸未定义 | 按比例从已知尺寸推算，在输出中标注"推测值" |
| 多文件描述同一部件 | 取最具体、最新的版本 |
| 某个部件太复杂无法建模 | 简化但不丢失识别度，注明简化了什么 |
| 渲染效果不理想 | 调 roughness/metalness/clearcoat，保持整体一致 |
| 不同版本之间有冲突 | 每个版本独立决策，不需要保持一致 |

### 8.3 High Agency 原则

- 你可以用任何工具：Blender、FreeCAD、OpenSCAD、Three.js、Fusion 360、Rhino 等等，不限
- 你有权决定每个版本用什么工具最合适（例如渲染版用 Blender Cycles，CAD 版用 FreeCAD，HTML 版用 Three.js）
- 遇到模糊地带，做出你认为最合理的选择并在交付报告中记录
- 同一部件可以尝试 2–3 种不同建模策略，比较效果后选最优
- 如果某个要求用常规方法实现效果很差，大胆探索替代方案

## 9. 推荐工作流（双钻模型）

### Phase 1: Discover

1. Clone 仓库：`git clone https://github.com/p-to-q/jiko.git && cd jiko`
2. 读完上面第 2 节列出的所有文件
3. 启动 `pnpm dev:web`，在浏览器里拖拽旋转查看当前实现
4. 拆解 `ShowcaseStage.tsx` 的 `buildHardware()`：理解每个部件怎么拼的、位置在哪、用了什么材质
5. 在脑子里把代码数值换算成 mm：usbW=0.19 → 8.34 mm，bodyW=1.82 → 80.1 mm

### Phase 2: Define

6. 输出一份你理解的**尺寸总表**（部件 → mm）
7. 决定每个版本用什么工具组合
8. 制定建模顺序（建议：机身 → 屏幕 → 背板 → 按钮 → 螺丝 → USB-C → 麦克风 → 散热标记）

### Phase 3: Develop

9. 建立统一的坐标系：原点在机身几何中心，Y 向上，Z 为厚度方向
10. 按部件顺序建模，每完成一个部件：
    - 导出 GLB
    - 导入 Three.js 场景（替换对应部件），检查比例
    - 调整到视觉匹配再进入下一个
11. 全部完成后再整体走一遍材质和光照调整

### Phase 4: Deliver

12. 按版本输出（先出 D，因为最精确→衍生出 A/B/C）
13. 每版自检对照质量清单
14. 撰写 `build-report.md`
15. 提交到仓库或交付给团队

## 12. 示例 Prompt 片段（可以直接复制给 AI Agent）

### 12.1 初始任务

> 你是一名资深 3D 硬件建模工程师。请克隆 https://github.com/p-to-q/jiko，阅读 `apps/web/src/ui/ShowcaseStage.tsx` 的 `buildHardware()` 和 `docs/hardware-phases.md` 的尺寸表，创建 Jiko One 的 3D 模型。我需要 4 个版本：A 渲染版（多子风格）、B HTML 网页查看器、C 3D 打印 STL、D 工业 CAD step。先读代码，再动手。

### 12.2 分步建模 — USB-C

> 请建 USB-C 接口模块。参考 `buildUsbCPort()`：外部凹坑 0.31×0.108、USB-C 口 0.19×0.052、舌头 0.12×0.015。单位换算 1 = 44 mm。舌头必须在接口内部偏下位置，接口底部必须是真正挖进去的。

### 12.3 分步建模 — 麦克风

> 请建顶部麦克风。参考 `buildMicAperture()`：沉孔半径 0.024（~1.06 mm）、内孔半径 0.016（~0.7 mm）。这是三层结构：顶部亮环 → 中间锥形沉孔 → 底部黑色圆孔。必须真挖，不是贴图。建完后用 30° 俯视图检查能否看见黑色内孔。

### 12.4 分步建模 — 散热孔

> 请建左侧平面上的散热孔（竖槽阵列）。参考 `buildVents()` 函数：4 条长竖槽 + 底部复合（短槽 + squircle 方孔）。槽宽 0.010 unit（0.44 mm），槽高 0.200 unit（8.8 mm），间距 0.022 unit（0.97 mm），底部方孔 0.035 unit（1.54 mm）。位置在左侧平面 `x = -(bodyW × 0.5) - 0.003`。填孔深色 + 边缘淡蓝灰高光。**打印版槽宽改为 0.8–0.9 mm**。

### 12.5 版本 A — 渲染子版本

> 渲染版需要出多个子版本：A1 暗夜金属（HDRI 环境光+高光倒角）、A2 半透视（外壳 40% 透明显示内部）、A3 胡桃木纹（参考 PILOT S20 风格）、A4 白金极简（通体铂金拉丝）、A5 自由发挥。每个子版本出 4K 渲染图（正 30°、左 45°、右 45°、底仰、顶俯）。屏幕凸起必须可见。

### 12.6 版本 B — HTML 查看器

> 请生成一个离线可用的 3D 查看器 HTML。单个文件，内嵌 Three.js（CDN 加载），包含 OrbitControls、版本切换下拉菜单（A1-A5 材质切换）、点击部件弹出标注。模型用 GLB 格式，可以 base64 内嵌或同目录加载。

### 12.7 版本 C — 3D 打印

> 请把 Jiko One 转为 3D 打印用的 STL。要求：水密、壁厚 ≥0.8 mm、装配间隙 0.2 mm、分件输出（body/button/screws/backplate）。打印方向不需要支撑。麦克风和 USB-C 孔要打通。附 print-notes.md 写明推荐打印参数。

### 12.8 版本 D — 最还原版

> 请输出最高精度的工业 CAD 版本。每个尺寸必须和 ShowcaseStage.tsx 代码逐行核对，偏差 ≤ 0.05 mm。输出 .step 格式，保留全部倒角，按装配层级组织部件。输出爆炸视图和尺寸校验 PDF。这是给工业设计师看的黄金参考。

### 12.9 质量自检

> 完成后，对照质量检查清单逐项自检：比例（80×120×6.6？USB-C 8.34？）、几何（麦克风通了吗？USB-C 有舌头吗？）、版本特有要求（HTML 能离线打开吗？STL 水密吗？）。不过关的不提交。
