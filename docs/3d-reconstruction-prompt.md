## 3. 核心尺寸规范

所有尺寸源自 USB-C 母座宽度（8.34 mm）。

```
scale = 8.34 mm / 0.19 model_unit ≈ 43.9 → 取整 44 mm / unit
```

### "热"字散热标记（左侧平面）

- 使用 `buildThermalMark()` 函数
- 位置：`x = -(bodyW / 2 + 0.004)`，`y = -bodyH × 0.26`（左下侧）
- 通过 Canvas 2D 渲染：扫描路径 → `Path2D` → 填充深棕 `rgba(45,42,38,1)` + 描边亮灰 `rgba(210,218,228,1)`
- 通过 `THREE.CanvasTexture` 贴在 `PlaneGeometry` 上
- 使用 `syncLeftFaceDepth()` 实现相机角度适配的深度测试

## 4. 建模要求

### 4.1 必须实打实建模，不是贴图

| 部件 | 要求 |
|---|---|
| **机身** | Squircle 形状主体，带倒角（bevel）。主体材料：黑色金属质感 |
| **屏幕** | 屏幕上内容要有 **物理凸起**（像素往上凸），在渲染图上必须能看出来 |
| **麦克风** | **真正挖出一个洞**。沉孔（countersink）+ 内孔（bore）+ 边缘高光环（chamfer）。不要用贴图模拟 |
| **USB-C** | **真正挖出接口**。包含：外部凹坑（recess）、内部 USB-C 口（inner mouth）、**USB 舌头（tongue）**、正面 rim、正面 mouth。舌头要在接口内部可见 |
| **螺丝** | 背面 4 颗真实螺丝几何，带螺丝槽 |
| **侧边按钮** | 按钮 + 导轨凹槽，与机身分离 |
| **散热标记** | "热"字图形在左侧平面，内凹黑底 + 亮灰描边 |
| **背板** | 机身背面较小的 Squircle 面板，与主体分离 |

### 4.2 材质

参考 `ShowcaseStage.tsx` 中的 `MeshPhysicalMaterial` 参数：

| 部件 | 颜色 | Metalness | Roughness | Clearcoat |
|---|---|---|---|---|
| 机身 | 0x010101 | 0.22 | 0.58 | 0.24 |
| 正面覆层 | 0x010101 | 0.20 | 0.60 | 0.20 |
| 背板 | 0x020202 | 0.26 | 0.60 | 0.20 |
| 螺丝 | 0xb8bcc4 | 0.88 | 0.22 | 0.28 |
| USB-C Rim | 0xc0c5cc | 0.88 | 0.20 | 0.30 |
| 按钮 | 0xc8c8d0 | 0.92 | 0.18 | 0.35 |
| 散热标记填充 | `rgba(45,42,38,1)` | 哑光 | 哑光 | 无 |
| 散热标记描边 | `rgba(210,218,228,1)` | 亮金属高光 | — | — |

### 4.3 输出格式

必须至少输出以下格式（可以并使用任意工具，包括但不限于 CAD 工具、Blender、Three.js、OpenSCAD 等）：

| 格式 | 用途 |
|---|---|
| **CAD (.step / .iges)** | 工程制造 |
| **GLB / glTF** | 网页渲染（Three.js 直接可用） |
| **STL** | 3D 打印 |
| **Three.js 源码（可选）** | 与当前 React 组件集成 |

### 4.4 屏幕凸起处理建议

屏幕像素/内容从机身表面微微凸起（0.1–0.3 mm 量级），参考方案：
- 在屏幕平面上方叠加一层极薄的物理起伏层（displacement / bump），对应 LED 像素位置
- 或者在 CAD 中将发光区域做 0.15 mm 的微浮雕
- 确保在斜面渲染图或近距离特写时肉眼可见

## 5. 冲突处理与决策原则

### 5.1 优先级

1. **视觉忠实度 > 制造可行性**（目前是工业设计展示阶段，不是量产图）
2. **与 ShowcaseStage.tsx 实现一致 > 重新发明**（代码实现是第一参考）
3. **多格式输出 > 单格式输出**
4. **迭代质量 > 一次性完成**

### 5.2 冲突时如何处理

| 冲突场景 | 处理方式 |
|---|---|
| 文档与代码不一致 | **以代码为准**（代码是最终执行的版本） |
| 某个尺寸未定义 | 按比例从已知尺寸推算，并在输出中标注该尺寸是推测值 |
| 多个文件描述同一部件 | 取最具体、最新的那个版本 |
| 某个部件太复杂无法用当前工具建模 | 简化但不丢失识别度，在文档中注明简化了什么 |
| 渲染效果不理想 | 调整材质参数（roughness / metalness / clearcoat），保持视觉风格一致 |

### 5.3 High Agency 原则

- 如果某个要求用常规方法实现效果差，请自由探索替代方案
- 你可以决定使用 Blender、FreeCAD、OpenSCAD、Three.js 或任何其他工具的任意组合
- 如果有更好的分割方式、更好的挤出策略、更好的 UV 方案，请大胆实施
- 当遇到模糊地带时，做出你认为最合理的选择并在文档中记录

**鼓励的策略：**
- 先用中低精度快速出第一版，检查比例和整体感
- 确认后再分部件精细化
- 同一部件可以尝试 2–3 种不同建模策略，比较效果

## 6. 推荐工作流（双钻模型，Double Diamond）

### Phase 1: 调研与发现（Discover）

1. Fork/clone 仓库：`https://github.com/p-to-q/jiko`
2. 阅读 Section 2 中列出的所有文件
3. 启动 `pnpm dev:web` 查看当前 Three.js 实现
4. 拆解 `ShowcaseStage.tsx` 中的 `buildHardware()`，理解每个部件的位置和尺寸

### Phase 2: 定义与规范（Define）

5. 导出所有部件的尺寸表（mm）
6. 标注所有部件的材料/颜色
7. 定义输出格式清单

### Phase 3: 开发与建模（Develop）

8. 建立统一的坐标系（原点在机身中心，Y 向上，Z 为厚度）
9. 分部件建模：
   - 机身 Squircle（含倒角 bevel）
   - 正面覆层 × 背板
   - 屏幕基板 + 像素凸起层 + 玻璃唇边
   - 右侧按钮（含导轨）
   - 背面 4 颗螺丝
   - 顶部麦克风（沉孔 + 内孔 + 边缘环）
   - 底部 USB-C（凹坑 + 接口 + 舌头 + Rim + Mouth）
   - 左侧"热"字散热标记
10. 每完成一个部件，导出 GLB 并叠加到当前 Three.js 场景中校验比例
11. 调整材质参数直到视觉匹配

### Phase 4: 交付与验证（Deliver）

12. 导出全部格式（CAD/GLB/STL）
13. 在 Three.js 场景中替换对应部件，确认位置和尺寸一致
14. 至少生成 3 个不同详细程度的版本：

| 版本 | 说明 |
|---|---|
| **v1 — 工业造型版** | 高精度 CAD，含全部倒角、拔模角、装配间隙。用于工程评估 |
| **v2 — 渲染展示版** | 优化三角面，高光材质，用于网页展示和营销渲染 |
| **v3 — 极简版** | 极低面数，保留轮廓识别度。用于快速迭代或 AR 预览 |

15. 每个版本附带检查清单，记录与参考实现的偏差

## 7. 质量检查清单（每次迭代必须自检）

### 比例
- [ ] 机身整体尺寸是否为 80 × 120 × 6.6 mm（±0.5 mm）
- [ ] USB-C 口宽度是否为 8.34 mm
- [ ] 微孔（螺丝/麦克风/USB-C 舌头）比例是否肉眼可见且合理

### 几何
- [ ] 麦克风是否真的挖通
- [ ] USB-C 是否有舌头内部结构
- [ ] 按钮是否与机身分离
- [ ] 散热标记是否位于左侧平面
- [ ] 倒角（bevel）是否平滑且对称

### 材质
- [ ] 机身是否为黑色金属质感（非纯黑，有反射）
- [ ] 散热标记填充色是否接近 `rgba(45,42,38,1)`
- [ ] 金属部件是否有适当的环境反射

### 输出
- [ ] 至少输出 CAD + GLB + STL
- [ ] GLB 在 Three.js 场景中替换后渲染一致
- [ ] STL 可被切片软件打开

## 8. 示例 Prompt 片段（你可以直接复制使用）

以下是为 AI Agent 准备的交流片段，适合在建模过程中反复使用：

### 8.1 初始任务

> 你是一名资深 3D 硬件建模工程师。请克隆仓库 https://github.com/p-to-q/jiko，阅读 `apps/web/src/ui/ShowcaseStage.tsx` 中的 `buildHardware()` 函数和 `docs/hardware-phases.md` 中的尺寸表，创建一个物理精确的 Jiko One 3D 模型。输出格式需要包括 CAD (.step)、GLB 和 STL。

### 8.2 分步建模

> 请先单独建 USB-C 接口模块。参考代码中 `buildUsbCPort()`：外部凹坑 width=0.31、USB-C 口 width=0.19、舌头 width=0.12。锚定单位制：1 = 44 mm。舌头必须在接口内部、位置偏下。

### 8.3 质量检查

> 请对刚建好的麦克风做自检：沉孔半径应约为 1.06 mm，内孔半径约 0.7 mm。在 3D 视口中从正面 30° 俯视检查是否能看见黑色内孔。如果看不到，说明建模方式有误，需要改为真正的环形挖孔。

### 8.4 冲突处理

> 文档 `docs/form-factor.md` 和代码 `ShowcaseStage.tsx` 对机身圆角的描述可能不同。当遇到冲突时，以代码为准。代码中 bodyRadius=0.15、exponentX=4.2、exponentY=3.8。

### 8.5 迭代要求

> 请按照双钻模型建模：先出粗糙版本验证比例，再分部件精细化，最后做多格式导出。每个版本结束前，对照质量检查清单逐项自检。不通过的不提交。

### 8.6 屏幕凸起

> 屏幕上的 LED 内容需要在渲染图上可以目测到凸起。建议做 0.15 mm 的微浮雕层覆盖屏幕区域。用 bump map 或 displacement 都可以，只要最终渲染能看出来。
